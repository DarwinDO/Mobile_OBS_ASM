# Dev Lifecycle

Use this skill for non-trivial mobile work.

## Default Flow

1. `new-requirement`
2. `review-requirements`
3. `review-design`
4. `plan`
5. `execute-plan`
6. `check-implementation`
7. `writing-test`
8. `capture-knowledge`

## Document Locations

- `docs/ai/requirements/`
- `docs/ai/design/`
- `docs/ai/planning/`
- `docs/ai/implementation/`
- `docs/ai/testing/`

## Rule

For simple one-file changes, use the same thinking but keep the docs minimal.
